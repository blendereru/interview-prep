## Iterator
C# provides so called methods called [iterator methods](https://learn.microsoft.com/en-us/dotnet/csharp/iterators#enumeration-sources-with-iterator-methods). This kind of method defines how to generate the objects in a 
sequence when requested. In other words, iterator produces a sequence of values lazily. This is achieved through the usage of 
`yield return` keyword. What lazy means is that the values are not produced immediately, but the execution pauses after each
`yield return` and resumes when the next value is requested.

Consider the following example:
```csharp
//1
public List<int> GetNumbers()
{
    var numbers = new List<int>();

    for (var i = 0; i < 1000000; i++)
    {
        numbers.Add(i);
    }
    return numbers;
}

//2
public IEnumerable<int> GetNumbersLazily()
{
    for (var i = 0; i < 1000000; i++)
    {
        yield return i;
    }
}
```
The first method creates 1 million integers in memory immediately. In the second method, values are produced one-by-one
when requested. So, eventually when user requests only the first 5 numbers, second method will only produce first 5, thus
making the request a lot faster:
```csharp
foreach (var number in comparison.GetNumbers().Take(5))
{
    Console.WriteLine(number);
}

foreach (var number in comparison.GetNumbersLazily().Take(5))
{
    Console.WriteLine(number);
}
```
`yield return` can be called multiple times in a method:
```csharp
public IEnumerable<int> GetSingleDigitNumbers()
{
    yield return 0;
    yield return 1;
    yield return 2;
    yield return 3;
    yield return 4;
    yield return 5;
    yield return 6;
    yield return 7;
    yield return 8;
    yield return 9;
}
```
This will produce the sequence of integers from 0 through 9 lazily. There is also a `yield break` keyword that can be used to
stop the sequence iteration. For example:
```csharp
IEnumerable<int> GetSingleDigitNumbersUpTo4()
{
    yield return 0;
    yield return 1;
    yield return 2;
    yield return 3;
    yield return 4;
    yield break;
    yield return 5;
    yield return 6;
    yield return 7;
    yield return 8;
    yield return 9;
}
```
Here, sequence stops after 4, so the rest `yield return` statements after `yield break` are ignored and can be omitted.

## Techinal implementation
What makes iterator methods generate values lazily is the state machine that is generated by the compiler.
Consider the example of what is generated in [GetSingleDigitNumber](https://sharplab.io/#v2:CYLg1APgAgTAjAWAFBQAwAIpwCwG5lQDMmM6AwugN7Lq2bFTboCyAFAJRU108BmA9gCcApgEMAxgAtWAN1GD0AOwCuAW3QBLRegDiwgC4BlLQHMANsIAiGkxv0A5NQCNhggM4dO1JD190sAJysKqrs+D5+6AC+3LQxEbSx9JhwhAA8WvoAfLoGxormVjZ2jqou7hxJ3pEpcJgA7Oio4TVYdVCNcC2RbQ3oMN1+vR3ohIO+w414STyT6ACs47Nw7Y0AbEv+K331m7RzABx7tX0B4/FRQA).
For the `GetSingleDigitNumbers` method, the compiler generates the class that acts as both the enumerable and the enumerator:
```csharp
[CompilerGenerated]
private sealed class <GetSingleDigitNumbers>d__1 : IEnumerable<int>, IEnumerable, IEnumerator<int>, IEnumerator, IDisposable
{
    private int <>1__state;

    private int <>2__current;

    private int <>l__initialThreadId;
    
    public C <>4__this;
    //...
}
```
Let's walk through the fields:
* `<>1__state` - state machine program counter. Is needed to transition between states. Whenever one `yield return` is executed, this `state` tracks where execution should resume after.
* `<>2__current` - current state value. `yield return 0` indicates `current` is 0, but state is 1(next value).
* `<>l__initialThreadId` - thread id where the iterator was created.
* `<>4__this`. Since iterator execution is suspended and later resumed,
the original stack frame no longer exists. Therefore, if the iterator accesses instance members, the compiler captures this 
into a field like `<>4__this` so the state machine can continue interacting with the original object across suspension points.

When the iteratar method(`GetSingleDigitNumbers`) is instantiated, internally the state  machine class is initialized with the current 
state of -2(indicating that the iterator is not started yet). The `initialThreadId` is set to the current thread id and the 
`__this` field is set to the original `this` object. In program, `GetEnumerator()` method is called then, which contains the following
conditions:
```csharp
IEnumerator<int> IEnumerable<int>.GetEnumerator()
{
    <GetSingleDigitNumbers>d__1 <GetSingleDigitNumbers>d__;
    if (<>1__state == -2 && <>l__initialThreadId == Environment.CurrentManagedThreadId)
    {
        <>1__state = 0;
        <GetSingleDigitNumbers>d__ = this;
    }
    else
    {
        <GetSingleDigitNumbers>d__ = new <GetSingleDigitNumbers>d__1(0);
        <GetSingleDigitNumbers>d__.<>4__this = <>4__this;
    }
    return <GetSingleDigitNumbers>d__;
}
```
Every call to `GetEnumerator()` method will return an independent iteration context, unless the state machine is not started yet. After
enumeration starts, subsequent calls return new independent state machine instances. `initialThreadId` is needed to protect
different thread from accessing the same state machine instance. State machine's state is then set to 0, indicating that
it is ready to start. Then, state machine calls `MoveNext()` method, which in our case is trivial. Every `yield return`
block is wrapped to each condition in `switch-case` condition:
```csharp
private bool MoveNext()
{
    switch (<>1__state)
    {
        default:
            return false;
        case 0:
            <>1__state = -1; 
            <>2__current = 0;
            <>1__state = 1;
            return true;
        case 1:
            <>1__state = -1;
            <>2__current = 1;
            <>1__state = 2;
            return true; 
        ///...
    }
}
```
As seen, during each iteration, state transitions to the next `yield return` block, and the current value is set to the
value returned by the `yield return` block. But this example does not show how important is `__this` field. 
So consider the [following](https://sharplab.io/#v2:CYLg1APgAgTAjAWAFBQAwAIpwCwG5lQDMmM6AwugN7Lq2bFTboCyAFAJRU108BmA9gCcApgEMAxgAt0rAG6jB6AHYBXALYAjYYoCWS5cIDu5fiqUAXba0LsAdGVMWO7bj1rUkbr5jgBOVqqa2uz4nt4Avq7okUgxBKQOZpaCyB48AA6COvKW6Hrm6AD6AM7mCuahUUQmSVb56KXlLmHuUTwlZYIFALwNnRVRcS30PoQAPPkAfDVOzTxp3lhwmADsRY1dod4+y1BrHeXoYOhwW4twu/sbBccwZ7QxMUA=)
example, where `MoveNext` looks like this:
```csharp
private bool MoveNext()
{
    switch (<>1__state)
    {
        default:
            return false;
        case 0:
            <>1__state = -1;
            <>2__current = <>4__this._start;
            <>1__state = 1;
            return true;
        case 1:
            <>1__state = -1;
            <>2__current = <>4__this._start + 1;
            <>1__state = 2;
```

Where `<>4__this` is actively used to capture the reference to the original `Counter` object, in which we access the 
`_start` field. So, you should always think that `<>4__this` is accessed whenever the iterator state machine 
needs to interact with the original instance object after the iterator method has been transformed into a separate
resumable object.

## Q & A
1) What happens when iterator returns a value besides int(which was most of our examples from) ?
* State machine class states operate on integer types, but the value of `<>2__current` will operate on type defines in iterator method.
[Example](https://sharplab.io/#v2:CYLg1APgAgTAjAWAFBQAwAIpwCwG5lQDMmM6AwugN7Lq2bFTboCyAFAJRU10/e0C+fepjiEAPFlQA+dAHEApgBcAyooBOASwB2AcwDOHIdSQ8eWOJgDs6AEQBBG/hOna5q7YBCjoYKT8gA==) is provided, 
where `<>2__current` is `string` type, but `MoveNext` transitions through `integer` states, where exactly as much states
as `yield return` blocks:
```csharp

{

    {
        default:
            return false;

            <>2__current = "B";
            <>1__state = 2;

        case 2:
            <>1__state = -1;
            return false;
    } 
}
```
